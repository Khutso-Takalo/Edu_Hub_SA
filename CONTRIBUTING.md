# Contributing to EduHub

Thank you for helping improve EduHub.

## Development Setup

1. Install dependencies.
2. Start the app.
3. Run tests before opening a pull request.

```bash
npm install
npm run dev
npm run test
npm run build
```

## Contribution Guidelines

- Keep changes focused and small.
- Add tests for behavior changes.
- Preserve verified-data policy: do not surface placeholder content as real data.
- For bursary data updates, prefer automated scraper output and verify links.

## Good First Issues

- Improve empty-state messaging.
- Add tests for data-quality filters.
- Expand classroom analytics visual summaries.
- Improve feedback export tooling.

## Pull Request Checklist

- Code builds successfully.
- Tests pass locally.
- New user-visible behavior is documented in README.
- No secrets or environment keys committed.
